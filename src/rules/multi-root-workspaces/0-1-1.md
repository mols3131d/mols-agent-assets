# Workspace Scopes & Boundaries

## Operational Scopes

| Term                  | Scope                                            | Visibility                                 |
| :-------------------- | :----------------------------------------------- | :----------------------------------------- |
| **에이전트(Agent)**   | Internal logic, latent space, thinking process   | **Private**: Agent only                    |
| **내부(Inner)**       | Communication path, session context, consensus   | **Confidential**: Investor & Agent         |
| **프로젝트(Project)** | Shared codebase, documentation, assets           | **Collaborative**: Team & Other Developers |
| **외부(Outer)**       | Consumers (Users), API server, broader ecosystem | **Public**: Users & External Systems       |

## Multi-root Directives

- **Scope Awareness**: Actions must be confined to the specified scope. Never leak `Agent` logic into `project` production code.
- **Root Isolation**: Personal assets in `Agent` must be strictly separated from `project` development roots.
- **Cross-Root Search**: If a target is missing in the current `project` root, proactively search all registered roots.
- **Execution Scopes**: Terminal commands and scripts must be executed within the specific `project` root relevant to the operation.
- **Path Validation**: Always verify the absolute path and its corresponding scope before performing critical operations.

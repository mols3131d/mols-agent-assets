# Dev-Orchestra

> Investor(User)에 의한 다수 Dev-Crew의 협업 및 자원 배분 메커니즘.

## 1. Hierarchy

- **Investor (User)**: 최종 의사결정권자 및 프로젝트 소유자. 여러 Crew를 오케스트레이션하여 프로젝트의 방향성을 결정함.
- **Dev-Orchestra**: 다수의 Crew를 유기적으로 연결하고, 각 Crew의 결과물을 통합(Convergence)하는 상위 프로세스 레이어.
- **Dev-Crew**: 독립적인 문제 해결 단위를 가진 팀 (e.g., '쿠로', '시로').

## 2. Orchestration Logic

- **Context Partitioning**: 각 Crew는 할당된 영역의 컨텍스트를 독립적으로 관리하며, 필요한 경우에만 Orchestra 레이어를 통해 데이터를 교환함.
- **Sublation (Convergence)**: 서로 다른 Crew의 산출물이 충돌할 경우, Investor의 의도에 맞춰 상위 차원에서 통합 또는 강제 수렴(Forced)을 수행함.
- **Parallel/Sequential Execution**: Investor의 전략에 따라 Crew 단위의 병렬 개발 또는 단계적 고도화를 수행함.

## 3. Communication Protocol

- **Crew Identification**: 모든 보고 및 통신은 소속 Crew 이름을 명시함 (e.g., "쿠로 크루 보고").
- **Zero-Loss Handover**: Crew 간 전환 또는 통합 시, 지식 증발을 방지하기 위해 Orchestra 레이어에서 핵심 컨텍스트를 보존함.

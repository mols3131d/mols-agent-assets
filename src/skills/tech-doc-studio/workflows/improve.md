# Improve Document

## Goal

Refactor, clarify, and enhance the quality of an existing document without changing its core meaning.

## When to Use

Use when the document needs readability improvements, grammar fixes, or structural refactoring.

## When NOT to Use

Do not use if the changes alter business or technical decisions (use update workflow).

## Instructions

- Read `references/<doc_type>.md` for best practices and anti-patterns.

## Workflows

### Arguments from Context

- Target file path

### Procedure

1. Review the existing document against best practices.
2. Refactor complex sentences, fix grammar, and improve readability.
3. Restructure sections to align with the recommended format if they are disorganized.

### Validation

- The document is easier to read and fully complies with the core methodology.

# 자산 명명 (Asset Naming)

## Goal (목표)

암묵적으로 기존 이름을 변경하지 않고 명확한 에이전트 자산 이름을 제안, 검증 또는 적용합니다.

## When to Use (사용 시점)

새 에이전트 자산에 적합한 이름을 결정하거나 명시적으로 요청을 받았을 때 기존 자산의 이름을 검증하고 적용하기 위해 이 워크플로우를 사용합니다.

## Instructions (지침)

- 명명 패턴과 예시는 `references/naming-convention.md`를 읽으십시오.
- 사용자가 명시적으로 요청하지 않는 한 기존 자산의 이름을 바꾸지 마십시오.

## Workflow: 자산 명명 (Asset Naming)

### Arguments from Context (컨텍스트 인자)

- 자산 유형 및 용도
- 기존 또는 새로운 자산 상태
- 이름 변경 적용 시 대상 경로

### Procedure (절차)

1. 이름 지정이 주된 요청인지 결정합니다.
2. 자산이 존재하는 경우 사용자가 이름 변경을 명시적으로 요청했는지 확인합니다.
3. `references/naming-convention.md`를 읽습니다.
4. 도메인과 작업을 식별하는 가장 짧은 이름을 제안하거나 검증합니다.
5. 명시적인 권한이 있을 때만 이름 변경을 적용한 다음 내부 참조를 업데이트합니다.

### Validation (검증)

- 스킬 디렉터리와 프론트매터의 `name`이 일치해야 합니다.
- 이름은 소문자, 숫자, 단일 하이픈(-)만 사용해야 합니다.
- 이름 변경이 적용된 후 내부 경로가 올바르게 확인되어야 합니다.

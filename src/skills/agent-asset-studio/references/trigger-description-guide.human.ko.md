# 에이전트 자산 트리거 설명문 작성법 (Writing Agent Asset Trigger Descriptions)

> 트리거 설명문(Trigger Description)은 기능의 요약본이 아니라, 실행 여부를 판별하는 의도 분류기(Intent Classifier)입니다.

## Purpose (목적)

`description` 필드는 에이전트 자산이 활성화될지 여부를 결정합니다. 기능의 단순 소개가 아닌, 사용자 요청에 대한 분류 기준(Classification Criteria)을 작성해야 합니다.

- 트리거 범위가 너무 좁으면, 필요한 상황에 스킬이 실행되지 못하는 오류(False Negative)가 늘어납니다.
- 트리거 범위가 너무 넓으면, 굳이 필요 없는 상황에 스킬이 불필요하게 켜지는 오류(False Positive)가 늘어납니다.

## Principles (원칙)

1. `USE WHEN...` 으로 문장을 시작합니다.
2. 에이전트의 내부 구현 방식이 아닌, 사용자의 최종 목표와 의도(Goal and Intent)를 기술합니다.
3. 정확한 키워드가 명시되지 않은 암시적인 요청들까지 `USE WHEN`에서 다룹니다. 별도의 설명이 보강 차원에서 명확한 구분에 도움이 되는 경우에만 `INCLUDES` 필드를 추가하십시오.
4. 처리 영역(Scope)과 명시적 제외 영역(Exclusions)을 모두 선언합니다.
5. 핵심만 간결하게 작성하여 1024자 이내로 구성합니다.

## Recommended Template (권장 템플릿)

```text
USE WHEN: [핵심 목표, 대표적인 작업 유형, 그리고 암시적 요청들].
INCLUDES: [필요한 경우에만 작성하는 추가 보완 설명].
EXCLUDES: [제외 대상 범위 및 유사하지만 처리하지 않는 상황].
```

## Example (작성 예시)

```text
USE WHEN: 사용자가 구조화된 데이터를 분석하거나, 메트릭의 변화를 조사하거나, 집단을 비교하고, 정량적 통찰을 이끌어내거나, 수치 하락/급증/성능 격차의 배후에 있는 요인을 식별하고자 할 때.
EXCLUDES: 분석 목적이 없는 파일 변환, ETL 구현 또는 데이터베이스 관리 작업.
```

## Checklist (체크리스트)

- [ ] 서술형 트리거문으로 시작하는가
- [ ] 사용자의 의도(Intent)에 중점을 두고 있는가
- [ ] 사용자의 암시적 요청까지 포괄하는가
- [ ] `INCLUDES`는 `USE WHEN`이 담지 못한 추가 정보를 제공할 때만 사용했는가
- [ ] 처리 범위와 제외 영역이 명확하게 기술되어 있는가
- [ ] 유사하거나 인접한 다른 스킬들과 경계를 구분하고 있는가
- [ ] 전체 1024자 이내의 분량을 유지하고 있는가

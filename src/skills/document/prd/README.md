# PRD Skill

This skill supports the agent in managing Product Requirement Documents (PRD) to define the product vision, goals, user scenarios, and feature requirements before implementation.

## Purpose
- Clearly define "What & Why" to build from the user and business perspective.
- Facilitate collaboration and alignment among PMs, designers, developers, and QA engineers.
- Manage active and archived PRDs systematically.

## Structure
- `SKILL.md`: Agent execution instructions (High-Density English)
- `templates/`: Includes PRD main index, archive index, and templates

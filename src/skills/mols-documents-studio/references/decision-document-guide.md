# 간이 결정 문서(Decision Document) 작성 가이드

이 문서는 에이전트 자산 및 프로젝트 아키텍처에 대한 의사 결정 내용을 기록하는 간이 결정 문서의 포맷과 작성 지침을 정의합니다.

## 파일 위치 및 규칙

- **위치**: `docs/skills/<skill-name>/decision.md`
- **목적**: 아키텍처 결정 사항, 제약 조건, 설계 의도와 그에 따른 영향을 한눈에 파악

## 포맷 구조

기본 구조는 아래 헤더와 목록 형식을 따릅니다.

```markdown
# Decisions (또는 결정 기록)

## 확정 (또는 Accepted)

### [결정 주제/제목]

- [구체적인 의사 결정 내용 설명]
- **Reason:** [이 결정을 내린 핵심 이유/동기]
- **Impact:** [이 결정으로 인해 시스템, 워크플로우, 혹은 개발자에게 미치는 영향]
```

## 작성 원칙

1. **간결성**: 세부 구현 코드 나열 대신 정책과 영향 중심 기술
2. **인과관계 명확화**: **Reason**과 **Impact**를 누락 없이 1:1로 매칭
3. **상태 관리**: 보류 중인 제안은 `## Proposed`, 대체된 결정은 `## Superseded` 섹션으로 분류

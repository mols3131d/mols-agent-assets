# Get Exit Workflow (Meta-Workflow)

이 워크플로우는 프로젝트의 요구사항과 에이전트 작업 관행을 분석하여, 작업 종료 시 에이전트가 수행해야 할 정리 및 핸드오프 절차를 정의한 `exit.md` 워크플로우를 생성합니다.

## `exit.md`란?

`exit.md`는 에이전트가 현재 세션을 종료하기 직전 수행해야 하는 '핸드오프(Handoff)' 절차를 정의한 워크플로우입니다. 작업 결과의 저장, 태스크 상태 업데이트, 중요한 의사결정 로그 기록 등을 통해 다음 세션(또는 다른 에이전트/개발자)이 맥락을 완벽하게 이어받을 수 있도록 보장하는 것이 목적입니다.

## Steps

### 1. **HANDOFF PROTOCOL DISCOVERY**

- 에이전트의 작업 기록 및 상태 보존 방식을 분석하여 다음을 파악합니다:
    1. **태스크 관리 방식**: 작업 진척도를 기록하는 파일의 위치와 업데이트 규칙.
    2. **컨텍스트 저장소**: 컨텍스트, 로그, 아티팩트 보관 등 에이전트의 사고 과정이나 중간 결과물을 저장하는 위치.
    3. **최종 보고 요구사항**: 사용자에게 보고해야 할 필수 항목
- **만약 작업 종료 및 정리 절차에 필요한 기본 정보를 파악하지 못했다면, 해당 사실을 사용자에게 보고하고 워크플로우를 종료합니다.**

### 2. **Create Exit Workflow**

- **맞춤형 생성**: 파악된 정보를 바탕으로 다음 단계를 포함하는 `exit.md`를 생성합니다.
- **저장 및 업데이트**:
    - 별도 지침이 없다면 **루트(Root)**를 기본 저장 위치로 선정하고 사용자에게 알립니다.
    - 기존 파일이 존재할 경우, 파일명 뒤에 `_`를 붙여 생성하고(예: `exit_.md`) 사용자에게 이를 경고합니다.

### 3. **REPORT**

- 작업 완료 후 사용자에게 다음 사항을 보고합니다:
    - **정의된 종료 프로토콜**: 에이전트가 작업을 마칠 때 수행할 주요 체크리스트.
    - **생성된 워크플로우의 특징**: 해당 프로젝트에서 맥락을 보존하기 위해 특히 강조된 정리 절차.

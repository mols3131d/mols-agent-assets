# 실행 계획: 워크플로우 가이드라인 정의 - 한국어 번역본

> **참고**: 이 문서는 사용자의 이해를 돕기 위한 번역본입니다. 에이전트는 원본(`implementation_plan.md`)을 기준으로 동작합니다.

`mols-agent` 저장소 내에서 고품질 에이전트 워크플로우를 작성하기 위한 표준과 원칙을 정의합니다.

## 사용자 검토 필요 사항

> [!IMPORTANT]
> - **워크플로우 구조**: 워크플로우는 상징적인 단계 표시(예: `[SCAN]`, `[EXEC]`)와 루트 상대 경로 링크를 우선적으로 사용합니다.
> - **중복 제거(DRY) 전략**: 가이드라인은 로직을 중복해서 작성하는 대신, 기존 자산에 대한 `@references` 또는 직접 링크 사용을 강조합니다.

  - "포맷 표준" 명시: 루트 상대 경로 링크(`/<dir>/<file>`), 최소한의 프론트매터, 상징적 헤더 사용.

## 검증 계획

### 수동 검증

1. 가이드라인이 사용자의 피드백(링크 스타일, 참조를 통한 DRY, 일반 문서와의 구분)을 모두 포함하고 있는지 확인합니다.
2. 상징적 헤더(예: `[SCAN]`)를 엄격하게 강제할 것인지, 아니면 권장 패턴으로 남겨둘 것인지 사용자에게 확인합니다.

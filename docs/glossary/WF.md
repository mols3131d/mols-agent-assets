# WF (Workflow)

## Definition

Dynamic specification of logical steps and task sequencing required for an agent to achieve a specific objective.

## Structure

- **Trigger**: Activation condition.
- **Step (Task)**: Atomic execution unit.
- **Node (Decision)**: Branching logic based on conditions.
- **Goal (Result)**: Terminal state or artifact generation.

## Usage

- Location: `{.agents}/workflows/`.
- Format: High-density, command-executable instructions.

## Philosophy

Automation of reasoning through pre-defined sequence graphs.

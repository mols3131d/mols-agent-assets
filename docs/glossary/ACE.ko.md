# ACE (Agent Context Engineering)

**에이전트 컨텍스트 엔지니어링(ACE)**은 에이전트의 사고 방식과 행동 지침을 단순한 텍스트 프롬프트가 아닌, 구조화되고 관리 가능한 **알고리즘적 자산**으로 설계하는 지식 공학 프로세스입니다.

## 개념 정의

- **본질**: 에이전트가 특정 상황에서 최적의 판단을 내릴 수 있도록 '뇌' 역할을 하는 컨텍스트(Rules, Workflows, Protocols)를 엔지니어링하는 것.
- **차별점**: 일회성 프롬프트 작성에서 벗어나, 지속적인 개선(Improvement)과 평가(Evaluation)가 가능한 지식 자산으로서의 접근을 지향함.

## 프레임워크 내 역할

- **Brain of Agent**: 에이전트가 작업을 수행할 때 참조하는 최상위 의사결정 체계.
- **Asset Management**: 모든 룰과 워크플로우는 버전 관리가 가능한 자산으로 취급됨.

## 핵심 가치

1. **재사용성**: 한 번 정의된 컨텍스트는 다른 에이전트나 프로젝트에서도 활용 가능.
2. **검증 가능성**: 구조화된 명세를 통해 에이전트의 행동이 예상대로 작동하는지 테스트 가능.
3. **진화 가능성**: 피드백 루프를 통해 지속적으로 지능의 밀도를 높임.

# 🤝 관행 (Convention)

이 폴더는 프로젝트에서 권장되는 **작업 습관, 관행(Practices), 그리고 스타일 선호도(Soft Rules)**를 정의합니다.

## 📌 핵심 원칙

- **일관된 습관**: 표준(Standard)이 강제적인 기술 명세라면, 관행은 협업의 흐름을 부드럽게 만드는 공유된 습관입니다.
- **실용성**: 실제 작업 과정에서 얻은 통찰을 바탕으로 "우리는 보통 이렇게 일한다"는 점을 명문화합니다.
- **유연성**: 상황에 따라 더 나은 방식이 있다면 논의를 통해 지속적으로 진화시켜 나갑니다.

## 📂 주요 관행 영역

- (현재 프로젝트 성숙도에 따라 지속적으로 업데이트 예정)
- 예: 에이전트 대화 톤앤매너, 코드 리뷰 관행, 폴더 네이밍 관습 등

_우리는 좋은 관행을 공유함으로써 명문화된 규칙 그 이상의 유대감과 작업 효율을 실현합니다._

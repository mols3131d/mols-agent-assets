# RFC (Request for Comments)

RFC는 새로운 기능 도입이나 대규모 리팩토링 전, 팀원들의 피드백을 받기 위해 작성하는 제안서입니다.

## 특징

- **목적**: 아이디어의 기술적 타당성 검토 및 팀 내 합의 도출.
- **주요 구성**: 제안 배경, 상세 설계, 예상되는 부작용, 대안 분석.
- **특징**: 활발한 토론이 중심이며, 논의 결과에 따라 내용이 수정되거나 기각될 수 있습니다.

---

## 의사결정 프로세스

1. **발의**: 새로운 아이디어를 RFC 초안으로 작성.
2. **검토**: 팀원들과 기술적 허점이나 대안을 논의.
3. **승인**: 논의 끝에 최선의 방안 확정.
4. **기록**: 확정된 설계를 산출물로 기록하여 보존.

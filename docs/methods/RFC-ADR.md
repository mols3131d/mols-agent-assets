# RFC and ADR

## RFC (Request for Comments)

- **Goal**: Technical feasibility and consensus.
- **Content**: Background, Design, Side-effects, Alternatives.
- **Status**: Volatile. Subject to change/rejection based on discussion.

## ADR (Architecture Decision Record)

- **Goal**: Preserve context of architectural decisions.
- **Content**: Decision, Context, Status, Consequences.
- **Status**: Immutable. Superseded by new ADRs if overturned.

## Workflow

1. **Draft**: Create RFC.
2. **Review**: Discussion & Feedback.
3. **Approve**: Reach consensus.
4. **Record**: Extract core design into ADR.
5. **Implement**: Execute based on ADR.

## Summary

RFC = Design (The Quest) | ADR = Record (The Fact)

# PRD and SRS

## PRD (Product Requirements Document)

- **Focus**: What & Why (Business Value).
- **Subject**: Background, User Persona, KPI, High-level Features.
- **Lead**: PM.

## SRS (Software Requirements Specification)

- **Focus**: How (Technical Logic).
- **Subject**: Data Model, API, Architecture, Performance, Constraints.
- **Lead**: Tech Lead / Architect.

## Workflow

1. **Market/Idea** (Discovery)
2. **PRD** (Definition)
3. **Tech Review** (Analysis)
4. **SRS** (Realization Logic)
5. **Dev & QA** (Validation)

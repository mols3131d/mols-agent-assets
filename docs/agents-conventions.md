# AI Agent File Conventions

이 문서는 AI 에이전트(Agent)가 프로젝트의 맥락을 빠르고 정확하게 파악하고, 일관된 방식으로 작업을 수행할 수 있도록 돕는 **파일 및 디렉토리 규칙(Conventions)**을 정리합니다.

## `AGENTS.md`

에이전트의 페르소나, 코딩 스타일, 금지 사항 등을 정의하는 파일들입니다.

- `README.md`와 `CONTRIBUTING.md`가 사람 개발자를 위한 것이라면, 이 두 파일의 내용을 기본으로 하되 에이전트에게만 추가로 지시하거나 강조해야 할 사항들을 `AGENTS.md`에 적습니다.
- `README.md`와 마찬가지로, `AGENTS.md`는 **계층적 지침(Hierarchical Guidelines)** 구조를 따릅니다. 루트의 지침을 기본으로 상속하되, 특정 하위 폴더의 맥락이 다를 경우 해당 폴더 내에 `AGENTS.md`를 작성하여 상위 지침을 **오버라이딩(Overriding)**하거나 상세한 로컬 지침을 제시할 수 있습니다.

## `.agents/`

에이전트의 자율적인 작업과 컨텍스트 유지를 위한 전용 공간입니다. 에이전트 전용 자산을 격리하여 관리함으로써 프로젝트의 노이즈를 줄이고 에이전트의 추론 효율을 높입니다.

| 디렉토리/파일    | 역할                                                                                                                               |
| :--------------- | :--------------------------------------------------------------------------------------------------------------------------------- |
| **`rules/`**     | **에이전트 전용 준수 사항**: 특정 기술 스택 적용 방식, 에이전트용 코딩 표준 등 에이전트가 작업 시 상시 참조해야 할 정적 규칙 문서. |
| **`workflows/`** | **표준 작업 절차**: 복잡한 태스크(커밋, 리팩토링, 검증 등)를 에이전트가 실수 없이 완수할 수 있도록 정의한 단계별 가이드.           |
| **`brain/`**     | **컨텍스트 지속성**: 에이전트가 내린 결정 배경, 분석 데이터, 세션 간 유지해야 할 '기억'을 기록하는 공간.                           |

---

> [!TIP]
> 요구 사항 기술서(PRD)나 기술 설계서와 같은 사양서는 에이전트 전용 폴더가 아닌, 인간 개발자와 공유하는 **`docs/`**와 같은 공용 디렉토리에서 관리하는 것이 도구와 인간이 동일한 베이스라인을 공유할 수 있어 더 효율적입니다.

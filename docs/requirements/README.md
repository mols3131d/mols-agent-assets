# SYNOPSIS

ACE_Requirements ::= {Rule_Specifications | Workflow_Protocols | Evaluation_Criteria}

Role ::= {Product_Requirements_Document (PRD) | Operational_Requirements_Document (ORD)}

# PROTOCOL

- **Context_as_Spec**: This folder serves as the definitive specification for how agents must think and act.
- **Requirement_Driven**: Every ACE asset must fulfill the requirements defined in this directory.
- **Continuous_Validation**: Requirements are living documents, updated as agent performance data evolves.

# TOPOLOGY

- rule/ :: Specifications for engineering and managing the lifecycle of behavioral rules.
- workflow/ :: Operational requirements for executing procedural agentic assets.

# CONSTRAINT

- Core_Objective: Define High-Density Requirements for Superior Agent Output.
- Compliance: ACE assets must adhere to the specs defined herein.

# ⚙️ 규칙 관리 (Rule Management)

이 폴더는 단순히 규칙을 나열하는 곳이 아닙니다. 규칙이 어떻게 만들어지고, 어떻게 더 나은 방향으로 개선되며, 그 가치를 어떻게 평가할 것인지에 대한 **관리 체계**를 정의합니다.

## 📌 핵심 패러다임: "Active Management"

- **저장이 아닌 관리**: 규칙은 정적인 문서가 아니라, 지속적으로 진화하는 프로젝트의 핵심 자산입니다.
- **가치 중심**: 좋은 규칙은 에이전트의 판단력을 높이고, 예외 상황을 줄이며, 결과물의 밀도를 높입니다.
- **수환 구조**: 생성(Creation) → 적용(Execution) → 개선(Improvement) → 평가(Evaluation)의 끊임없는 순환 과정을 지향합니다.

## 📜 문서 구성 정보

- **[핵심 관리 표준 (rule.md)](./rule.md)**:
    - 엘리트 규칙의 정의 (무엇이 좋은 규칙인가?)
    - 규칙 엔지니어링 프로토콜 (어떻게 만드는가?)
    - 개선 및 진화 루프 (어떻게 발전시키는가?)
    - 평가 지표 (어떻게 성공을 측정하는가?)

_우리는 완벽한 규칙을 한 번에 만드는 것이 아니라, 지속적인 관리를 통해 완벽에 가까운 규칙으로 진화시켜 나갑니다._

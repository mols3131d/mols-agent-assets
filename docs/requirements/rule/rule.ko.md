# ⚙️ 규칙 라이프사이클 관리 (Rule Lifecycle Management)

이 문서는 `mols-agent` 생태계 내에서 규칙의 전체 라이프사이클을 관리하기 위한 전문 표준을 정의합니다. 우리는 단순히 규칙을 저장하는 것이 아니라, 그 진화를 엔지니어링합니다.

## 📜 명세 상세 내용

### 1. "엘리트 규칙"의 정의 (품질 표준)
고품질의 규칙은 다음 기준을 충족해야 합니다:
- **논리 밀도**: 최소한의 토큰으로 최대한의 정보를 전달합니다. 상징과 구조화된 정의를 사용합니다.
- **실행 가능성**: 에이전트가 즉시 따를 수 있는 명확한 지침이나 제약을 제공해야 합니다.
- **원자성**: 개선 과정에서의 부작용을 방지하기 위해 하나의 규칙은 하나의 논리적 개념이나 경계만 다룹니다.
- **일관성**: `/docs/standards/agent.md` 및 기타 루트 레벨 정책과 일치해야 합니다.

### 2. 생성 프로토콜 (규칙 엔지니어링)
새로운 규칙을 초기화할 때:
1. **요구사항 분석**: 현재 규칙 세트의 공백을 식별합니다. 중복을 피합니다.
2. **초안 작성 (TAS 접근법)**:
    - **정(Thesis)**: 초기 규칙 구조 제안.
    - **반(Antithesis)**: 예외 케이스와 잠재적 에이전트 실패로 규칙에 도전.
    - **합(Synthesis)**: 유연성과 엄격함의 균형을 맞추어 규칙 개선.
3. **검증**: `/studio/` 내의 특정 시나리오에 대해 규칙을 테스트합니다.

### 3. 개선 및 진화 (루프)
규칙은 살아있는 자산입니다. 실행 로그와 실패 분석을 바탕으로 개선되어야 합니다.
- **정밀화**: 에이전트 작업 중 발견된 모호한 언어를 명확히 합니다.
- **최적화**: 장황한 설명을 기호(`::=`, `|`, `{}`)로 교체하여 논리 밀도를 높입니다.
- **통합**: 패턴이 나타나면 관련 하위 규칙을 하나의 프레임워크로 통합합니다.

### 4. 평가 지표 (QA)
규칙의 성공을 측정하는 방법:
- **준수율**: 복잡한 작업에서 에이전트가 규칙을 올바르게 적용한 비율.
- **환각 감소**: 규칙이 알려진 실패 모드를 성공적으로 방지했는지 여부.
- **논리적 명확성**: 다른 에이전트(또는 인간)가 모호함 없이 의도된 행동을 재현할 수 있는지 여부.

## 🔄 동기화 가이드
- 이 문서의 모든 논리적 변경 사항은 대응하는 영어 명세(`.md`)와 일치해야 합니다.
- 규칙의 실행 로직은 영어 명세를 최종 권위로 삼습니다.

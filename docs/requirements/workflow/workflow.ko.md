# 🔄 워크플로우 라이프사이클 관리 (Workflow Lifecycle Management)

`mols-agent`의 워크플로우는 정적인 지침이 아니라, **실행 가능한 알고리즘 자산**입니다. 이 문서는 에이전트의 성능을 극대화하기 위해 워크플로우의 전체 라이프사이클을 관리하는 표준을 정의합니다.

## 📜 명세 상세 내용

### 1. 품질 표준 (고가치 워크플로우의 기준)
워크플로우는 다음의 고밀도 기준을 충족하도록 엔지니어링되어야 합니다:
- **실행 가능성**: 모든 단계는 인간의 개입 없이 에이전트가 실행할 수 있을 만큼 명확해야 합니다.
- **구조적 무결성**: `[SCAN] -> [PLAN] -> [EXEC] -> [VERIFY]` 순서 또는 특수 변형을 따라야 합니다.
- **논리 밀도**: 산문을 최소화하고 상징적 지시어와 구조적 마커를 최대화합니다.
- **DRY (중복 배제)**: 기존 규칙(`@rule`)과 하위 워크플로우를 활용하여 논리 중복을 피합니다.

### 2. 엔지니어링 및 디자인 (생성)
1. **범위 정의**: 정확한 입력, 프로세스 및 예상 출력(산출물)을 정의합니다.
2. **모듈형 설계**: 복잡한 작업을 원자적이고 재사용 가능한 단계로 나눕니다.
3. **상징적 매핑**: 표준 헤더를 사용하여 작업 유형을 표시합니다:
    - **`[SCAN]`**: 환경 및 상태 정찰.
    - **`[PLAN]`**: 전략 합성 및 매핑.
    - **`[EXEC]`**: 자산의 실제 변경 또는 생성.
    - **`[VERIFY]`**: 품질 보증 및 논리 검증.

### 3. 최적화 루프 (개선)
워크플로우는 실행 텔레메트리 및 병목 현상을 바탕으로 진화해야 합니다.
- **마찰 감소**: 에이전트가 자주 멈추거나 설명을 요청하는 단계를 식별하여 단순화합니다.
- **정밀도 튜닝**: `[PLAN]`과 `[VERIFY]`의 언어를 날카롭게 다듬어 출력 품질의 편차를 줄입니다.
- **자동화 (Turbo)**: 매우 안정적이고 자동 실행이 안전한 단계에만 `// turbo` 주석을 추가합니다.

### 4. 평가 기준 (성공 지표)
- **성공률**: 수동 수정 없는 "첫 번째 성공" 빈도.
- **사이클 타임**: `[SCAN]`부터 `[REPORT]`까지의 작업 완료 효율성.
- **토큰 효율성**: 최소한의 지침으로 목표를 달성하는 정도.

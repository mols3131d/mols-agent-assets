# Document Studio (document-studio)

`document-studio`는 프로젝트 내 워크스페이스 문서(ADR, PRD, SPEC, 설계서, 태스크 리스트, 칸반 등)를 효율적으로 생성, 갱신, 검증 및 인덱싱하기 위한 문서 관리 통합 스킬입니다.

이 문서는 개발자와 기획자가 문서를 일관성 있게 작성하고 관리하기 위해 `document-studio` 스킬과 그 CLI 도구들을 활용하는 방법을 설명합니다.

## 개요 및 역할

`document-studio`는 정보의 파편화와 검색 오버헤드를 막기 위해 **라우팅 구조**를 사용합니다.
스튜디오 내 진입점인 `SKILL.md`가 모든 개별 문서 가이드를 내포하는 대신, `sub-skills/INDEX.csv` 파일을 참조해 사용자의 요청(예: "ADR 작성해 줘", "PRD 업데이트해 줘")에 맞는 세부 문서화 지침으로 연계(Routing)해 줍니다.

### 핵심 설계 원칙

- **KISS**: 하나의 문서는 명확한 하나의 목적을 가져야 합니다.
- **DRY**: 동일한 단일 진실 공급원(Single Source of Truth)을 여러 문서에 중복 기술하지 않습니다.
- **행동 가능성(Actionable)**: 모든 문서는 담당자(Owner), 상태(Status), 날짜(Date) 등의 정보를 메타데이터로 명시하여 실행 가능한 상태로 관리합니다.

## 작성 및 관리 흐름 (Workflow)

에이전트 또는 작업자가 문서 작업을 진행할 때 아래의 절차를 따릅니다.

1. **상대 문서 유형 식별 및 라우팅**:
   - `sub-skills/INDEX.csv`를 읽고, 사용자의 목적(ADR, PRD, SPEC 등)에 맞는 세부 하위 스킬을 선정하여 로드합니다.
2. **템플릿 활용 문서 초기화**:
   - 직접 문서를 빈 파일에서 작성하는 대신, 스튜디오의 초기화 스크립트(`init_document.py`)를 통해 사전에 준비된 표준화된 마크다운 템플릿 구조를 기반으로 문서를 자동 생성합니다.
3. **기존 문서 백업**:
   - 기존 문서를 재작성하거나 대규모 편집을 수행하기 전, 안전을 위해 기존 파일명을 `<파일명>.original.md`로 임시 백업합니다.
4. **인덱스 및 상태 자동 갱신**:
   - 새 문서가 생성되거나 문서의 상태가 변경(예: Draft -> Approved 등)되면 인덱스 갱신 스크립트(`update_index.py`)를 통해 폴더 로컬 `INDEX.csv`를 자동으로 최신 상태로 갱신합니다.

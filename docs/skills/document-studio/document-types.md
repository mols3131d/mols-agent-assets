# Document Studio - 지원하는 문서 유형 (Document Types)

`document-studio`가 다루는 주요 문서 유형과 각각의 용도 및 작성 규칙은 다음과 같습니다. 이 가이드라인들은 `sub-skills/` 디렉토리 내에 개별 스킬 파일로 저장되어 정적 라우팅을 지원합니다.

---

## 1. ADR (Architecture Decision Record)
아키텍처 결정 사항에 대한 기록입니다.

- **용도**: 아키텍처 상의 트레이드오프, 특정 기술이나 라이브러리 선정 배경 및 결정 내역을 기록합니다.
- **포함 사항**: 배경(Context), 결정 사항(Decision), 예상되는 영향(Consequences), 현재 상태(Status: Proposed, Accepted, Rejected 등).

---

## 2. PRD (Product Requirements Document)
제품 또는 기능의 요구사항에 관한 문서입니다.

- **용도**: 해결하려는 비즈니스 문제, 유저 스토리, 기능의 목표 및 핵심 기능 목록을 기술합니다.
- **포함 사항**: 제품 목표(Product Goal), 사용자 스토리(User Stories), 기능 요구사항(Feature Needs), 제외 사항(Exclusions) 등.

---

## 3. SPEC (Implementation Specifications)
실제 구현을 위한 기술 스펙 문서입니다.

- **용도**: PRD 요구사항을 바탕으로 구체적인 데이터 모델, API 인터페이스, 비즈니스 로직 설계 및 시스템 구조를 정의합니다.
- **포함 사항**: API 스키마, 데이터 저장 계획, 컴포넌트 간 흐름도 등.

---

## 4. Design Notes (설계서 및 디자인 메모)
UX/UI 및 화면/시스템의 세부 디자인 설명입니다.

- **용도**: 사용자 인터페이스 흐름, 와이어프레임 링크, 화면 구성 요소, 상태 변경 메커니즘을 상세히 기록합니다.

---

## 5. Tasks (태스크 리스트)
실행 가능한 작업 목록입니다.

- **용도**: 특정 이슈나 프로젝트 내에서 당장 수행해야 하는 작업 항목을 체크리스트 형태로 관리합니다.
- **포함 사항**: `[ ]` (미완료), `[/]` (진행중), `[x]` (완료) 상태를 사용합니다.

---

## 6. Kanban Board (칸반 보드)
워크스페이스 내 작업 단계 현황을 한눈에 보여주는 테이블 형식의 보드입니다.

- **용도**: 각 작업 카드들의 현황(To Do, In Progress, In Review, Done)을 관리합니다.
- **수행 방식**: 칸반 스크립트를 사용하여 카드를 이동시키거나 상태를 업데이트합니다.

---

## 7. Index (인덱스 레지스트리)
폴더 로컬 자산 및 문서의 인덱스를 관리하는 파일(`INDEX.csv`)입니다.

- **용도**: 폴더 내 문서들의 프런트메터(Frontmatter) 정보를 자동으로 읽고 가공하여 메타데이터와 파일 목록을 갱신합니다.

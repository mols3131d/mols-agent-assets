# Iceberg Vision

## 소개

Iceberg는 **빙산의 일각**이다. 전체 정보 중 현재 필요한 일부만 보여주고, 나머지는 숨기거나 생성하지 않는다.

Iceberg는 **인간을 위한 스킬**이다. 정보가 많을 때 핵심과 다음 탐색 경로를 제공해 사용자가 쉽게 이해하고 행동하도록 돕는다.

## 문제 정의

모든 정보를 한 번에 보여주면 다음 문제가 발생한다.

- **정보 쓰나미**: 방대한 정보가 한꺼번에 쏟아지는 현상
- **이해 부채**: 소화하지 못한 정보가 누적되는 상태
- **이해 병목**: 복잡성 때문에 판단과 행동이 멈추는 상태

## 정보 공개 방식

정보를 공개 시점, 깊이, 위치에 따라 단계적으로 제공한다. 처음에는 핵심만 보여주고, 상세 정보는 필요할 때 탐색하게 한다.

### 1. 핵심 정보 제공

목적 달성에 필요한 정보만 최소 1문장에서 최대 3문단으로 먼저 제공한다. 부연 설명과 디테일은 2번 또는 3번으로 분리한다.

### 2. 엔드포인트 제공

상세 내용 대신 다음 탐색 경로를 제공한다.

- 힌트, 팁, 제안, 단서
- 링크, 엔드포인트
- 관련 문서와 참조 경로

### 3. 정보 미생성 또는 별도 문서화

정보를 삭제하는 것이 아니라 현재 작업의 흐름에서 분리하는 방식이다.

- 요청되지 않은 세부 정보는 생성하지 않는다.
- 큰 설명은 별도 문서, 부록, 후속 산출물로 분리한다.
- 현재 판단에 필요 없는 배경 정보는 숨긴다.

## 핵심 원칙

1. **핵심 우선**: 목적 달성에 필요한 최소 정보부터 제공한다.
2. **점진적 공개**: 상세 정보는 필요에 따라 단계적으로 연다.
3. **탐색 경로 제공**: 숨긴 정보에 접근할 힌트와 링크를 남긴다.
4. **불필요한 생성 억제**: 요청되지 않은 설명과 산출물을 만들지 않는다.
5. **이해 가능성 우선**: 완전성보다 이해와 다음 행동을 우선한다.
6. **맥락 존중**: 사용자 단계와 목적에 따라 공개 수준을 조절한다.

## 기대 효과

- 필요한 정보를 쉽게 찾고 빠르게 이해한다.
- 정보 과잉으로 인한 판단 지연을 줄인다.
- 필요한 시점에만 상세 정보로 들어간다.
- 정보 쓰나미, 이해 부채, 이해 병목을 예방한다.

## 간접 효과

- 에이전트 컨텍스트 감소: 점진적 공개, 링크, 임베드, 라우팅으로 필요한 컨텍스트만 제공한다.

## 성공 기준

사용자가 현재 필요한 정보를 받고, 다음 탐색 경로를 알며, 과도한 설명 없이 작업을 계속할 수 있으면 Iceberg가 적용된 것이다.

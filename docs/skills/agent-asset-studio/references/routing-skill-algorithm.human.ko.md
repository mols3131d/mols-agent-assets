# 라우팅 스킬 알고리즘 (Routing Skill Algorithm)

라우팅 스킬의 표준 선택 방식, 모호성 해결 및 로딩 동작 사양입니다.

## Router Template (라우터 템플릿)

```markdown
## Routing (라우팅)

1. `workflows/INDEX.csv`를 단 한 번 읽습니다.
2. 요청된 최종 결과(Outcome), 작업 종류(Operation), 대상 객체(Object) 및 제약 사항을 파악합니다.
3. `excludes` 조건과 일치하는 경로(Routes)를 제외합니다.
4. `use_when` 조건과 일치하는 최소한의 경로 세트를 선택합니다.
5. 의미상의 실질적인 모호함이 있다면, 하나의 명확한 질문을 던져 해결합니다.
6. 인덱스 디렉토리를 기준으로 선택된 파일들의 ID를 찾아내고, 해당 파일들만 로드합니다.
7. 선택된 워크플로우가 요구하는 경우에만 참조된 공유 리소스를 로드합니다.
8. 완료하기 전에 선택된 각 워크플로우의 검증(Validation) 단계를 실행합니다.

키워드 중복이 아닌 핵심 의도(Semantic intent)를 바탕으로 라우팅해야 합니다. 경로를 찾기 위해 `workflows/` 디렉토리를 통째로 스캔하지 마십시오.

## Ambiguity (모호성 처리)

- 요청한 사항을 완전히 충족하는 경로가 있다면 단 하나의 경로만 선택합니다.
- 사용자의 요청이 명시적으로 여러 경로에 걸쳐 있을 때만 여러 경로를 한 번에 선택합니다.
- 남아 있는 경로들이 서로 전혀 다른 작업을 수반할 때만 하나의 타겟 질문을 통해 모호성을 해결합니다.
- 매칭되는 경로가 전혀 없는 경우, 해당 스킬이 사용자의 요청을 지원하지 않음을 알립니다.
```

## Selection Rules (선택 규칙)

- 긍정 조건(`use_when`)을 비교하기 전에, 부정 조건(`excludes`)을 강력한 부정 신호로 삼아 먼저 제외 처리합니다.
- `use_when`은 구체적인 예시 및 암시적 의도를 바탕으로 매칭합니다. 키워드는 동점 상황을 해결할 때만 보조적으로 사용합니다.
- 요청 전체를 포괄할 수 있는 가장 작은 크기의 경로 세트를 선호합니다.
- 각 `id`는 `INDEX.csv` 파일이 위치한 디렉토리를 기준으로 상대 경로를 해석합니다.

## Loading Rules (로딩 규칙)

- 라우팅의 깊이는 단 1개 레이어만 유지해야 합니다. 워크플로우가 재귀적으로 다른 워크플로우를 라우팅하게 만들지 마십시오.
- 선택된 워크플로우와 해당 워크플로우가 명시적으로 요구하는 리소스만 로드합니다.
- 파일의 정확한 ID 지정을 디렉토리 스캔(Scan) 방식으로 대체하지 마십시오.
- 사용자의 명시적인 요청이나 절차상 요구사항이 있는 경우에만 워크플로우의 우선순위/실행 순서를 유지합니다.

# Agent Asset Studio - 하위 스킬 (Sub-skills)

`agent-asset-studio`는 특정 자산 관리 태스크를 보다 구체적으로 처리하기 위해 다음과 같은 하위 스킬(Sub-skills)들을 운영하고 있습니다. 이 하위 스킬들은 `sub-skills/` 디렉토리 하위에 위치하며, 각각 고유의 시나리오를 가집니다.

---

## 1. agent-skill (`sub-skills/agent-skill.md`)
에이전트 스킬 자산의 생성, 개선, 평가 및 검증을 담당하는 핵심 하위 스킬입니다.

- **목적**: 하나의 작업만 수행(One skill = one job)하며, LLM 비용을 최소화하고 가독성을 극대화한 스킬 파일을 만듭니다.
- **주요 규칙**:
  - **KISS/DRY/YAGNI**를 적용하여 불필요한 빈 디렉토리나 중복 규칙 생성을 금지합니다.
  - 핵심 가이드라인은 `SKILL.md`에 배치하고, 상세 내용이나 참조 자료는 별도의 `references/` 하위 마크다운으로 분할하여 컨텍스트 윈도우 낭비를 막습니다.
- **검증 게이트**:
  - Frontmatter 필수 필드 검사 (이름은 kebab-case, 길이 등)
  - 본문 길이 경고 (최대 500라인 준수 여부)

---

## 2. asset-naming (`sub-skills/asset-naming.md`)
에이전트 자산들의 파일명 및 내 명칭이 프로젝트 규칙을 준수하는지 확인하고 교정하는 스킬입니다.

- **네이밍 규칙**:
  - **일반 스킬 (Standard Skills)**: `<도메인>-<동사>-<상세정보>.md` 형식 (예: `create-agent-skill.md`)
  - **라우팅 스킬 (Routing Skills)**: `<도메인>-<상세정보>-<위치>.md` 형식 (예: `agent-asset-studio.md`, 여기서 위치는 `console`, `studio` 등)
- **수행 방식**:
  - 잘못 설정된 자산명을 교정하거나, 일관된 스타일 가이드를 제시하여 네이밍 중복과 파편화를 예방합니다.

---

## 3. compress (`sub-skills/compress.md`)
장기적인 에이전트 대화 및 실행 세션에서 발생하는 입력 토큰 비용을 줄이기 위해, 비대해진 에이전트 자산 파일의 본문을 핵심 의미를 유지한 채 압축하는 스킬입니다.

- **원작 보존**: 압축을 수행하기 전 반드시 원본 파일을 `<파일명>.original.md`로 백업해 둡니다.
- **압축 대상**: 불필요한 서술어, 장황한 설명, 중복되는 예시 등 의미 전달에 지장이 없는 텍스트를 제거하여 압축율을 극대화합니다.

---

## 4. routerize-skills (`sub-skills/routerize-skills.md`)
관련성이 높거나 기능이 세분화된 여러 개의 스킬들을 하나의 마스터 라우팅 스킬과 `INDEX.csv` 구조로 병합하여 구조화하는 스킬입니다.

- **적용 대상**: 동일한 도메인 내에 유사한 작업을 수행하는 스킬이 3개 이상 늘어났을 때, 이를 마스터 라우팅 스킬 하위로 통합하여 인덱싱 비용을 효율화합니다.
- **출력물**: 라우터 역할을 하는 `SKILL.md` 및 하위 스킬 레지스트리인 `INDEX.csv`를 자동으로 구성합니다.

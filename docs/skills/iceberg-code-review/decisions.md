## Accepted

| Decision | Reason | Impact |
| :--- | :--- | :--- |
| **[인코딩] 문서 파일은 UTF-8 사용**: 리뷰 문서와 템플릿을 UTF-8로 읽고 씀 | **문자 표현 일관성**: 한국어와 기호를 환경에 관계없이 보존 | **모든 문서 I/O에 `encoding="utf-8"` 적용** |
| **[리뷰 전 검증] summary 파일 복사 후 리뷰 범위만 테스트**: 리뷰 시작 전에 리뷰 대상 범위의 테스트를 실행 | **기존 실패와 리뷰 영향 분리**: 리뷰 기준 상태를 먼저 확인 | **테스트 범위를 리뷰 범위와 일치**: 전체 리뷰는 전체 테스트, 부분 리뷰는 관련 테스트만 실행 |
| **[폴더명] 생성 시각 포함**: 리뷰 폴더명을 `yyyy-MMdd-HHmm-<title-slug>` 형식으로 생성 | **같은 날 생성한 리뷰 구분**: 시간과 분으로 폴더명 충돌을 줄임 | **생성 시점의 로컬 날짜·시간 사용** |
| **[책임 범위] 리뷰 문서 포맷에 집중**: 리뷰 결과의 표현과 문서 구조만 담당 | **리뷰 관점과 출력 형식 분리**: 방향성·고도화 판단은 별도 전문성에 위임 | **다른 스킬·컨텍스트와 조합**: 리뷰 자체의 전략을 중복 정의하지 않음 |
| **[검증] 프론트 매터 외 주석 검사 유지**: HTML 주석을 검사 | **HTML 주석의 명확한 경계**: 문서 내용과 안정적으로 구분 가능 | **잔여 템플릿 지침 차단**: HTML 주석이 남으면 validator가 실패 처리 |
| **[문서 작성] 템플릿 주도**: placeholder와 주석으로 작성 절차 제공 | **작성 절차의 단일 출처**: 작성 규칙을 템플릿에서 함께 관리 | **문서별 워크플로우 불필요**: 별도 작성 워크플로우를 추가하지 않음 |
| **[스크립트] 결정적 작업 위임**: 반복 가능한 작업을 스크립트로 처리 | **모델 비용 절감**: 결정적 작업을 모델 컨텍스트에서 분리 | **판단 작업에 모델 집중**: 모델은 추론이 필요한 작업에 사용 |
| **[스크립트] 최소 검증 지원**: 구조와 필수 조건을 검사 | **객관적 오류의 저비용 탐지**: 결정적으로 판정 가능한 오류만 자동 검사 | **스크립트와 모델의 책임 분리**: 의미 판단은 모델이 처리 |
| **[스크립트] 생성 로직 지원**: 경로, 파일명, 템플릿 복사를 처리 | **반복 절차의 결정성**: 같은 입력에 같은 생성 결과 제공 | **출력 편차 감소**: 모델의 반복 작업을 제거 |
| **[스크립트] 오버 엔지니어링 금지**: 필요한 최소 자동화만 구현 | **자동화의 총비용 비교**: 구현·유지 비용과 모델 처리 비용을 함께 고려 | **모델이 유리하면 자동화 생략**: 더 단순하거나 저렴한 처리 방식 선택 |
| **[문서 구조] 추가 섹션 허용 설정 지원**: `allow_extra_sections`로 허용 여부를 선택 | **확장성과 엄격도 선택**: 프로젝트별 추가 섹션 요구를 config로 제어 | **기본값 `true`**: 추가 섹션을 기본 허용하고, `false`이면 필수 섹션만 허용 |
| **[프론트 매터] 추가 필드 허용 설정 지원**: `allow_extra_frontmatter`로 허용 여부를 선택 | **metadata 확장성과 엄격도 선택**: 프로젝트별 추가 필드 요구를 config로 제어 | **기본값 `true`**: 추가 필드를 기본 허용하고, `false`이면 필수 필드만 허용 |
| **[포맷] rumdl 후처리 시도**: 검증 성공 후 Markdown 포맷 적용 | **일관된 문서 형식**: 반복 가능한 포맷 작업을 도구에 위임 | **실패 비차단**: rumdl 부재나 실행 실패는 경고만 기록 |
| **[Config] 사용자 설정 지원**: 사용자별 동작 설정을 별도 파일로 관리 | **환경별 요구 분리**: 공통 스킬 수정 없이 설정 변경 | **개인 설정 보존**: 저장소와 분리된 `user_data/config.json` 사용 |
| **[Config] 문서 저장 경로 지정**: `reviews_dir`에서 리뷰 저장 위치 설정 | **경로 변경의 단일 지점**: 생성 명령마다 경로를 반복 지정하지 않음 | **Workspace별 저장 위치 지원**: 상대 경로를 workspace 기준으로 해석 |

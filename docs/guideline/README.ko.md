# 🧭 가이드라인 (Guideline)

이곳은 **에이전트 컨텍스트 엔지니어링(Agent Context Engineering, ACE)**을 위한 종합 가이드라인을 정의하는 공간입니다. 

단순히 에이전트가 지켜야 할 '행동 지침'을 나열하는 수준을 넘어, **에이전트의 지능을 결정하는 핵심 자산(규칙, 워크플로우 등)을 어떻게 설계하고, 관리하며, 지속적으로 진화시킬 것인가**에 대한 방법론을 다룹니다.

## 📌 핵심 목적: "Context as an Engineered Asset"

- **자산화 및 체계화**: 규칙과 워크플로우를 소모성 문서가 아닌, 정교하게 설계된 '엔지니어링 자산'으로 취급합니다.
- **매니지먼트 표준**: 고품질의 에이전트 컨텍스트를 생성(Create), 평가(Evaluate), 개선(Improve)하기 위한 통일된 절차를 제공합니다.
- **성능의 극대화**: 더 높은 수준의 추론과 작업을 수행할 수 있도록 컨텍스트의 밀도와 정확성을 지속적으로 고도화합니다.

## 📂 주요 가이드라인 영역

- **`rule/`**: 에이전트의 판단 기준과 제약 사항(Rules)의 라이프사이클 관리 방법론
- **`workflow/`**: 에이전트의 실행 절차(Workflows)에 대한 설계 표준 및 최적화 프로세스

_우리는 에이전트를 단순히 '사용'하는 것을 넘어, 에이전트의 사고와 행동의 근간이 되는 **컨텍스트를 엔지니어링**함으로써 프로젝트의 가치를 창출합니다._

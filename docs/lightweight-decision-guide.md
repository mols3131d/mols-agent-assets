# 간이 결정 문서 작성 규칙

여러 프로젝트에서 재사용할 수 있는 간이 결정 문서 형식이다. 기본은 상태별 표이며, 표로 부족한 결정만 별도 헤더로 보강한다.

## 기본 규칙

- 결정은 `제안`, `채택`, `대체`, `폐기` 상태별 `##` 헤더 아래에 둔다.
- 표에는 `결정`, `이유`, `영향` 열만 둔다.
- 번호와 ID는 사용하지 않는다.
- 한 행에는 하나의 결정만 기록한다.
- 상태가 바뀌면 기존 행을 새 상태 표로 옮긴다.
- 짧은 결정은 표에서 끝낸다.

## 상태 정의

| 상태 | 의미 | 사용 시점 | 다음 처리 |
|---|---|---|---|
| 제안 | 검토 중이며 아직 적용을 확정하지 않은 결정 | 선택지와 영향 검토가 남았을 때 | 채택하거나 폐기한다. |
| 채택 | 적용하기로 확정한 현재 결정 | 합의가 끝났거나 구현 기준으로 사용할 때 | 유지하거나 새 결정으로 대체한다. |
| 대체 | 다른 결정이 대신해 더 이상 현재 기준이 아닌 결정 | 방향, 제약, 근거가 바뀌어 새 결정으로 교체했을 때 | 새 결정을 채택 상태에 기록한다. |
| 폐기 | 채택하지 않거나 적용을 중단한 결정 | 필요성이 사라졌거나 비용·위험이 이점보다 클 때 | 필요해지면 새 제안으로 다시 기록한다. |

## 상태별 표

```markdown
# 간이 결정 문서

## 제안

| 결정 | 이유 | 영향 |
|---|---|---|
| 결정 내용 | 선택을 검토하는 이유 | 예상되는 변화 |

## 채택

| 결정 | 이유 | 영향 |
|---|---|---|
| 결정 내용 | 선택한 이유 | 적용 후 변화 |

## 대체

| 결정 | 이유 | 영향 |
|---|---|---|
| 대체된 결정과 새 결정 | 변경한 이유 | 마이그레이션 또는 호환성 영향 |

## 폐기

| 결정 | 이유 | 영향 |
|---|---|---|
| 폐기한 결정 | 폐기한 이유 | 제거하거나 되돌릴 범위 |
```

빈 상태는 헤더와 표를 만들지 않는다.

## 상세 기록

맥락, 대안, 제약, 후속 작업을 표 한 행으로 설명하기 어려울 때만 같은 상태 아래에 상세 기록을 추가한다.

```markdown
## 상태명

### 결정 제목

#### 맥락

결정이 필요해진 배경과 문제를 적는다.

#### 결정

선택한 내용을 적는다.

#### 이유

이 선택이 문제에 맞는 이유를 적는다.

#### 영향

얻는 점, 비용, 제약을 적는다.

#### 대안

검토했지만 선택하지 않은 대안이 있을 때만 적는다.
```

상세 기록이 여러 문단으로 커지거나 독립적인 검토 이력이 필요하면 개별 문서로 분리하고 상태별 표에서 링크한다.

# Human-Agent Asset Pipeline (휴먼-에이전트 자산 파이프라인)

## Goal (목표)

사람을 위한 가독성(자세한 한국어 설명)과 에이전트를 위한 효율성(압축된 영어 지시문) 사이의 모순을 해결하기 위해, 에이전트 자산에 소스 코드(Source)와 컴파일(Compile) 개념을 도입합니다.

## When to Use (사용 시점)

새로운 에이전트 자산(스킬, 레퍼런스, 워크플로우 등)을 설계하고 작성할 때 이 파이프라인을 적용합니다.

## Instructions (지침)

### 1. Source 파일 (`file.human.md`)

- **작성 언어 및 형태**: 한국어로 상세한 배경, 설명, 사람이 읽기 편한 문장 구조로 작성합니다.
- **저장 위치**: 작성 중이거나 완성된 후에도 `workbench/` 또는 사람을 위한 `docs/` 디렉토리에 보관됩니다. `src/` (에이전트 실사용 디렉토리)에는 절대 이동시키지 않습니다.
- **용도**: 자산의 진실 공급원(Single Source of Truth)이자 사람이 직접 읽고 유지보수하는 '소스 코드' 역할을 합니다.

### 2. Compiled/Binary 파일 (`file.md`)

- **작성 언어 및 형태**: `.human.md`의 내용을 에이전트가 읽기 편하게 극도로 압축된 영어(Caveman style 등)로 자동 변환한 결과물입니다.
- **저장 위치**: 컴파일 및 검증이 끝나면 최종적으로 `src/`로 이동하여 에이전트의 실제 컨텍스트로 사용됩니다.
- **용도**: 에이전트의 컨텍스트 창(Context window) 비용을 아끼기 위한 '실행 파일(Binary)' 역할을 합니다.

## Constraints (엄격한 제약 사항)

- **단방향 동기화 유지**: `.human.md` 원본이 존재하는 자산의 경우, 결과물인 `.md` 파일을 절대로 직접(수동으로) 수정해서는 안 됩니다.
- **수정 절차**: 변경 사항이 생기면 반드시 원본 `.human.md`를 먼저 수정한 후, 에이전트를 통해 `.md`로 다시 압축(컴파일)하는 단방향 빌드 프로세스를 거쳐야 합니다. 이를 어기면 소스와 실행 파일 간의 내용이 불일치하게 됩니다.

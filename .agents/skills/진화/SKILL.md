# 규칙 진화 (Rule Evolution)

> 동적 맥락 캐싱 및 운영 패턴 로깅을 위한 고밀도 프로토콜.

> **진화 로그 (EVOLUTION_LOG)**: `.agents/brain/evolution.md`

## 1. 패턴 로깅 및 통계적 캐싱

1.  **트리거 (Trigger)**: 에이전트가 반복되는 오류, 사용자의 직접적인 교정 또는 비효율적인 운영 루프를 통계적으로 인식함.
2.  **평가 및 조치**:
    - **IF `[기존 패턴]`**: 델타(Delta) 분석 -> 캐싱된 제약 조건 정제 -> **로그 업데이트**.
    - **ELSE `[신규 패턴]`**: 트리거 원인 분석 -> 임시 소프트 제약 초안 작성 -> `신규 자율 가설`에 **추가** (무관한 경우 폐기).

## 2. 실행 프로토콜

아래 구조를 엄격히 준수하여 `EVOLUTION_LOG`를 업데이트합니다. 이 항목들을 절대적 진리가 아닌 임시적인 행동 가중치로 취급하십시오.

```markdown
# 규칙 진화 (Rule Evolution)

## 공식 규칙 권장 사항 (Recommended for Official Rules)

1. **키워드**: 지침 / 제약 사항

## 신규 자율 가설 (New Autonomous Hypotheses)

1. **키워드**: 지침 / 제약 사항
```

- **공식 규칙 권장 사항**: 핵심 `.agents/rules/` 통합을 위해 사용자에게 제안된, 통계적으로 검증된 패턴.
- **신규 자율 가설**: 사용자의 의도에 휴리스틱하게 정렬하기 위해 현재 및 향후 세션에 캐싱된 임시 소프트 제약.

## 3. 생명 주기 및 유지 관리

1.  **생성 (캐싱)**: 새로 식별된 패턴은 반드시 `신규 자율 가설`에 추가되어야 함.
2.  **성숙**: 가설이 여러 세션에 걸쳐 통계적으로 유효함(오류/교정 감소)이 입증되면 `공식 규칙 권장 사항`으로 이동.
3.  **공식화**: 사용자가 권장 규칙을 `.agents/rules/`에 공식적으로 통합하면 `EVOLUTION_LOG`에서 해당 항목을 **삭제**.
4.  **정리 트리거 (FIFO)**: 새로운 제약 조건을 추가할 때 총 개수를 확인. **IF > 30**, 맥락 저하를 방지하기 위해 `신규 자율 가설`에서 가장 오래되거나 관련성이 낮은 캐시 항목을 삭제.

## 4. 맥락 인식 분류 (전략적 필터링)

1.  **신규 패턴 진입**: 현재 세션에서 식별된 모든 신규 패턴은 기본적으로 `신규 자율 가설`에 입력되어야 함.
2.  **보편적 vs 특수적**:
    - **보편적 패턴**: 프로젝트/아키텍처 전체에 적용 가능한 원칙(예: 보안, 에러 핸들링)은 안정화 후 `공식 규칙 권장 사항`으로 제안될 수 있음.
    - **맥락 특화 패턴**: 특정 작업(예: HTML-to-MD, 특수 CI 스크립트)에 최적화된 로직은 공식 규칙으로 권장해서는 안 됨. 유사 작업 반복 시 맥락을 제공하기 위해 가설 상태로 유지.
3.  **성숙도 검사**: 다양하고 복수적인 맥락에서 통계적으로 유용성이 입증된 패턴만 `권장` 상태로 격상.

# 보고서 작성 스킬

> 내부 보고서 작성 스킬

```yaml
dist_path: "<Inner Workspace>/reports/<MM-DD>/" # 내부 보고서 폴더 경로
file_format: "<00>--<Author>--<Keyword>.md" # 보고서 파일 이름 형식 ex) 01--쿠로-사키--code-review.md
```

---

## Steps

### Prepare

1. 목적 파악.
2. 맥락 탐색.
3. 저장 위치, 파일 이름 정의.

### Write

1. 빈 파일 생성.
2. 초안 작성.
3. 신뢰도: 논리적 결함이나 환각 여부를 검토.

### Edit

1. 가독성/가시성 개선.
2. 파일 이름 편집

### Brief

> 채팅으로 브리핑 한다.

1. 생성 위치
2. 요약

---

## Rules

- **근거**: 모든 결론은 증거(로그, 코드, 문서, 맥락)에 기반해야 합니다.
- **밀도 > 분량**: 불필요한 수식어를 배제하고 핵심 정보와 액션 아이템에 집중합니다.
- 가시성/가독성: 인간 친화적 문서

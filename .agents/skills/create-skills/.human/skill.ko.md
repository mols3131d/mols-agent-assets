# Create Skills

> Skill for Creating Skills

이 스킬은 새로운 에이전트 스킬(Skill)을 설계하고 생성할 때 준수해야 할 표준 절차와 형식을 정의합니다.

## Workflow

새로운 스킬을 생성할 때는 다음 순서에 따라 진행합니다.

### 1. 설계 및 트리거 정의

- 스킬이 해결하려는 핵심 문제를 파악합니다.
- 에이전트가 이 스킬을 어떤 상황에서 호출해야 하는지(`When to use`) 구체적으로 기획합니다.
- 이 내용은 스킬의 메타데이터(`description`) 작성의 기준이 됩니다.

### 2. 뼈대 생성

- 목적에 부합하는 직관적인 스킬 이름을 `kebab-case`로 명명합니다.
- `./templates/` 폴더를 복사하여 스킬 디렉토리를 생성합니다.
- 이를 통해 `SKILL.md`, `README.md`, `VERSION` 등 필수 파일이 자동으로 셋업됩니다.

### 3. 지침 작성 (`SKILL.md`)

에이전트가 직접 읽고 실행할 코어 지침을 작성합니다.

- **프론트매터**: `name`과 `description` 필드를 목적에 맞게 설정합니다.
  - 에이전트가 스킬을 발견하기 쉽도록 "언제 사용하는지"와 "무엇을 하는지"를 포함하여 간략히 작성합니다. (예: `<상황>에 사용합니다. <기능>을 수행하여 <결과>를 돕습니다.`)
  - 스킬 이름은 `kebab-case`를 사용하며, 폴더명과 `name` 필드를 일치시키십시오.
  - 스킬의 `description`은 에이전트가 해당 스킬을 선택할지 결정하는 핵심 데이터이므로 구체적으로 작성하십시오.

### 4. 가이드 작성 (`README.md`)

인간 개발자가 읽을 스킬 개요를 작성합니다.

- 스킬의 목적, 파일 구조, 활용 방법 등 인간이 이해하고 유지보수하는 데 필요한 가이드를 제공합니다.

### 5. 확장 및 리뷰

- 필요에 따라 스킬 구동을 돕는 보조 스크립트(`scripts/`)나 참고 자료(`references/`) 등을 구성합니다.
- 완성된 폴더 구조와 문서 초안을 유저와 검토하고, 피드백을 반영해 스킬을 최종 확정합니다 (Human-in-the-Loop).

## Directory Structure

```txt
[skill-name]/
├── SKILL.md          # [필수] 에이전트 실행 지침 및 메타데이터
├── README.md         # [필수] 스킬 개요 및 인간용 가이드
├── VERSION           # [필수] 버전 정보 (Semantic Versioning)
├── templates/        # [선택] 스킬 관련 문서 및 코드 템플릿
├── scripts/          # [선택] 자동화용 실행 스크립트
├── .human/           # [선택] 인간 가독성 전용 문서 (ko.md 등). 에이전트 접근 금지
└── references/       # [선택] 기술 명세 및 참고 자료
```

### `.human/`

이 폴더는 오직 인간 사용자를 위한 공간입니다. 에이전트는 이 폴더의 내용을 읽거나 수정하지 않아야 하며, 필요한 경우 `.aiexclude` 등을 통해 접근을 차단하십시오.

# 에이전트 자산 스크립트화 검토 (Evaluate Scriptification)

## Goal

특정 에이전트 자산이나 워크플로우 단계가 결정론적 스크립트로 변환 가능한지, 그리고 변환하는 것이 타당한지 트레이드오프와 ROI를 평가합니다.

## Non-Goal

- 실제 스크립트 코드 생성이나 구현 계획 수립.
- LLM의 추론(맥락 이해, 의도 파악, 모호성 해결)이 필요한 작업을 억지로 하드코딩된 스크립트로 밀어넣는 것.

## When to Use

사용자가 자연어 지침을 스크립트로 변환하자고 제안했을 때, 실제 계획이나 코딩을 시작하기에 앞서 그 적합성을 평가하기 위해 이 워크플로우를 사용합니다.

## When NOT to Use

- 사용자가 검토를 생략하고 즉시 계획 수립이나 실행으로 넘어가기를 명시적으로 요청한 경우 (단, 속으로 간단한 적합성 체크는 필수).
- 해당 작업이 본질적으로 유연한 텍스트 생성이나 LLM 추론을 요구하는 경우.

## Instructions

- 해당 작업이 LLM의 추론, 맥락 이해, 의도 해석 또는 유연한 텍스트 생성을 필요로 한다면 스크립트화를 즉시 중단하고 거부하십시오.
- 명확한 투자 대비 수익(ROI) 없이 불필요한 엔지니어링 복잡성만 가중시킨다면 스크립트화를 중단하십시오.
- `references/zen-of-agent-assets.md`를 읽어 추론(Reasoning)과 실행(Execution)을 명확하게 분리하십시오.
- 자연어만으로 충분하고 효율적인 작업이라면 코드로의 변환을 억지로 강요하지 마십시오.

## Workflows

### Arguments from Context

- 대상 워크플로우 또는 자산 경로
- 검토할 구체적인 단계나 절차

### Procedure

1. 스크립트화 대상 단계를 읽고 적합성을 분석합니다.
2. 추론 의존성을 확인합니다: 해당 단계가 숨은 맥락을 읽어내거나, 의미를 추출하거나, 비결정론적인 텍스트를 생성해야 합니까?
3. 추론이 필요하다면, 구체적인 이유를 들어 스크립트화를 반려(Recommend against)하고 워크플로우를 종료합니다.
4. 추론이 필요하지 않다면, ROI를 평가합니다: 스크립트가 자연어보다 훨씬 빠르고, 저렴하며, 신뢰할 수 있습니까? 그렇지 않다면 반려합니다.
5. 스크립트화가 적합하고 유익하다면, 긍정적인 검토 결과를 출력하고 `scriptify-plan` 워크플로우로 넘어갈 것을 권장합니다.

### Validation

- 평가 결과가 추론의 경계를 명확히 식별합니다.
- 출력 결과에 스크립트화 권장 여부와 그 이유가 명시되어 있습니다.

# 에이전트 자산 스크립트화 실행 (Apply Scriptification)

## Goal

승인된 스크립트화 계획에 따라 실제 스크립트를 작성하고, 대응하는 자연어 워크플로우를 업데이트하여 작업을 완료합니다.

## Non-Goal

- 스크립트화의 타당성 재평가.
- 사용자와의 협의 없이 승인된 계획을 대폭 수정하는 행위.

## When to Use

구현 계획이 확정되고 승인된 후, 실제 코드를 작성하고 마크다운 파일을 수정하기 위해 이 워크플로우를 사용합니다.

## When NOT to Use

- 대상 경로, 스크립트 언어, 대체할 정확한 섹션을 상세히 기술한 명확한 계획이 없는 경우.

## Instructions

- 깔끔하고, 결정론적이며, 독립적으로 실행 가능한 스크립트를 작성하십시오.
- 수정된 마크다운 워크플로우가 명시적인 구문(예: `run_command` 등)을 사용하여 스크립트를 호출하도록 보장하십시오.
- 기존 워크플로우의 원래 의도와 목표를 그대로 유지하십시오.

## Workflows

### Arguments from Context

- 승인된 구현 계획 (Implementation Plan)
- 대상 워크플로우 또는 자산 경로
- 대상 스크립트 경로 및 언어

### Procedure

1. 지정된 언어와 대상 경로를 바탕으로 스크립트를 작성하고 테스트합니다.
2. 스크립트가 의도한 모든 기계적 단계를 커버하며 정상 작동하는지 확인합니다.
3. 대상 마크다운 워크플로우를 업데이트합니다:
   - 스크립트를 호출하는 명시적인 명령어를 삽입합니다.
   - 더 이상 사용되지 않는 기존 자연어 지침을 제거합니다.
4. 업데이트된 마크다운 파일의 구조가 올바른지 확인하기 위해 표준 검증 절차(예: `validate_asset.py`)를 실행합니다.

### Validation

- 스크립트가 대상 경로에 존재하며 오류 없이 실행됩니다.
- 워크플로우가 스크립트를 올바르게 참조합니다.
- 스크립트 내부에 강제된 LLM 추론 로직이 없습니다.

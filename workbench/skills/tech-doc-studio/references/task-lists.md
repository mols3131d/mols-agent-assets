# Task Lists

## 1. Purpose (목적)

실행 가능한 작업 단위(Actionable work items)와 체크리스트를 관리하여 업무의 명확성을 높이고 집중력을 유지합니다.

## 2. Core Methodology (핵심 원칙)

- **Decompose**: 거대한 태스크("Mega-tasks")를 피하고, 며칠 내로 완료 가능한 명확한 하위 작업으로 잘게 쪼갭니다.
- **3-Bucket Framework**: 작업을 시작하기 전 목표(Goal), 결과물(Deliverables), 불확실성/가정(Uncertainties)을 명확히 정의합니다.
- **Granular & Current**: 세분화된 리스트를 최신 상태로 유지하여, 언제나 '다음으로 해야 할 가장 작은 행동'이 무엇인지 알 수 있게 합니다.

## 3. Recommended Structure (권장 구조)

- **[ ] To Do**: 명확한 액션 동사로 시작하는 대기 작업.
- **[/] In Progress**: 현재 진행 중인 작업 (단일 포커스 권장).
- **[x] Done**: 완료된 작업.
- **Sub-tasks**: 들여쓰기를 활용한 하위 세부 작업 체크리스트.

## 4. Best Practices (모범 사례)

- **Document as You Go**: 코딩 후 몰아서 문서화하는 것이 아니라, 문서화 자체를 태스크의 일부로 취급하여 진행 과정과 컨텍스트를 보존합니다.
- **Action-Oriented**: "데이터베이스"와 같은 명사형이 아닌 "데이터베이스 스키마 마이그레이션 스크립트 작성"과 같이 행동 지향적으로 작성합니다.
- **Clear Next Steps**: 완료된 항목은 빠르게 정리하거나 줄을 그어 항상 다음 스텝이 최상단에 오도록 유지합니다.

## 5. Anti-Patterns (안티패턴)

- 너무 추상적이거나 범위가 커서 어디서부터 시작해야 할지 모르는 태스크 (예: "로그인 기능 구현").
- 진행 상태가 전혀 업데이트되지 않아 실제 상황과 괴리된 리스트.

# 에이전트 규칙 (Agent Rules)

## 1. 핵심 추론 및 의도

- **추론**: 단순 수치보다 논리를 우선하며, 단순히 테스트를 통과하는 것이 아니라 설계 의도를 완벽히 충족해야 합니다.
- **의도**: 명확한 의도가 파악된 경우에만 행동합니다. 사용자의 의도를 함부로 짐작하지 않으며, 가정을 명시하거나 명확한 확인을 요청해야 합니다.
- **안정성**: 임시방편(Hack)을 지양합니다. 장기적인 무결성과 정밀한 수정을 최우선으로 합니다.

## 2. 미니멀리즘 및 단순성

- **KISS**: DRY(중복 방지)와 KISS(단순성 유지) 원칙을 준수합니다. 과잉 설계를 피합니다.
- **최소 실현 코드**: 문제 해결에 엄격히 필요한 최소한의 코드만 작성합니다.
- **리팩토링**: 더 단순한 해결책이 있다면 이를 제안하고 구현합니다.
- **정밀도**: 꼭 필요한 부분만 수정하며, 기존 코드 스타일을 반드시 준수합니다.

## 3. 실행 및 검증

- **검증 가능한 성공**: 태스크를 테스트 가능한 목표로 변환합니다. 변경 전후에 모든 테스트가 통과하는지 확인해야 합니다.
- **반복적 계획 수행**: 다단계 작업 시 [단계] -> [검증] 루프를 엄격히 준수합니다.
- **정리**: 본인의 작업으로 인해 생성되었거나 쓸모없어진 코드/임포트만 제거합니다.

## 4. 표준 및 규약

- **언어**: 사용자-에이전트 소통(한국어)과 기술적 자산(프로젝트 지침 준수)을 분리하여 소통의 정확성과 기술적 지속성을 보장합니다.
- **지침 상속**: 루트까지의 `README.md` 및 `AGENTS.md` 지침을 계층적으로 통합하여 최상위 원칙으로 준수합니다.
- **묵시적 수행**: 지침은 별도 선언 없이 즉각 실행하며, 부차적 설명 대신 완결된 결과물로 소통의 밀도를 극대화합니다.

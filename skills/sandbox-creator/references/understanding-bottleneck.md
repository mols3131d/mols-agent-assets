# Understanding Bottleneck & Solutions

> "Understanding is new bottleneck."

AI generate code fast. Dev role shift: write code → understand system. Keep cognitive ability, participate active.

## 1. Understanding Bottleneck & Cognitive Debt

Code verification automated. Human understand system for **participation**. Know concept structure → propose ideas, judge design.

**Cognitive Debt**: system work, human understand fail keep up.

1. AI generate code fast
2. Human understand fail, changes accumulate
3. Concept structure opaque
4. Human pushed out decision loop

Understanding bottleneck push human out loop.

## 2. Solutions

Use AI amplify human understand, not only generate code.

### Solution 1: Explain Diff & Quiz

Convert code diff to custom learn material.

- **Explain background**: explain goal + concept diff before code.
- **Literate Code Diff**: order code by human understand sequence, not physical file.
- **Quiz**: pace reader, prevent skip without understand.

### Solution 2: Micro World

Visual/empirical explore tool for internal behavior + peripheral context.

- **Features**: fast build, ephemeral software, visualize core behavior.
- **Examples**: timeline debugger, migration learn tool.
- Provide peripheral vision (state changes around bug).

### Solution 3: Shared Space

Scale understand from individual to team.

- **Collab log**: transparent record agent plan, human feedback, design debate.
- Process become shared asset. Lower onboard cost, raise team understand.

## 3. Conclusion

True bottleneck = **human understand**, not code volume. Use AI build learn/explore tools (explain diff, micro world, shared space). Help human grasp system + participate active.

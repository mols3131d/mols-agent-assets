# 자산 인덱스 작성 (Write Asset Index)

## Goal (목표)

라우팅 CSV, 독립형 마크다운 또는 기존 마크다운 파일 내부의 인덱스 표 형식으로 자산 인덱스를 생성하거나 업데이트합니다.

## When to Use (사용 시점)

에이전트 자산의 인덱스를 생성하거나 업데이트해야 할 때 이 워크플로우를 사용합니다. 인덱스는 라우팅 CSV, 독립형 마크다운 파일 또는 기존 파일 내부의 마크다운 표가 될 수 있습니다.

## Instructions (지침)

- `csv` 모드의 경우, 위치나 라우팅 의미론(semantics)을 선택하기 전에 `references/routing-skill-structure.md`를 읽으십시오.
- 마크다운 모드에는 참조가 필요하지 않습니다.
- 모드, 대상 또는 소스 범위가 누락되었고 안전하게 유추할 수 없는 경우, 구체적인 질문을 하나 던지십시오.
- 소유권이나 경계가 불분명한 기존 수동 작성 표를 대체하기 전에 중단하십시오.
- 소스 콘텐츠에 근거가 없는 경우, 라우팅 조건이나 문서 설명을 임의로 지어내지 마십시오.

## Workflow: 자산 인덱스 작성 (Write Asset Index)

### Arguments from Context (컨텍스트 인자)

- 모드(Mode): `csv`, `markdown-file` 또는 `markdown-table`
- 대상 인덱스 또는 대상 마크다운 경로
- 인덱싱할 파일 또는 워크플로우 모듈

### Procedure (절차)

1. 모드, 대상, 소스 범위 및 정렬 규칙을 확인합니다. 작성하기 전에 기존 콘텐츠를 검사합니다.
2. 명시적인 소스로부터 엔트리를 구축합니다. 상대 경로, 결정론적 경로 정렬 및 소스 제목이나 메타데이터에서 유도된 간결한 설명을 사용합니다.
3. 선택한 모드를 적용합니다:

   | 모드 | 출력 | 규칙 |
   | --- | --- | --- |
   | `csv` | `INDEX.csv` | `id,use_when,excludes` 사용. 기본값은 `workflows/INDEX.csv`. 인덱스 디렉터리를 기준으로 각 `id` 확인 |
   | `markdown-file` | `INDEX.md` | 제목과 링크 표 하나 추가. 유용한 소개말이나 수동 유지 관리 섹션 보존 |
   | `markdown-table` | 기존 `.md` 파일 | 무관한 내용을 다시 작성하지 않고 명확한 제목이 있는 링크 표 하나를 추가하거나 업데이트 |

4. `csv` 모드의 경우 한 행에 하나의 워크플로우만 유지합니다. 긍정적이고 암묵적인 선택 조건은 `use_when`에 넣고, 거의 일치하지만 제외되는 항목과 범위를 벗어나는 요청은 `excludes`에 넣습니다.
5. 마크다운 모드의 경우 설명이 포함된 링크 레이블과 상대 링크를 사용합니다. 인덱스 자체, 숨겨진 파일, 생성된 백업 및 무관한 아티팩트는 제외합니다.
6. 중복 항목을 추가하는 대신 기존 항목을 제자리에서 업데이트합니다. 사용자가 두 가지 모두를 요청하지 않는 한 CSV와 마크다운 인덱스를 동기화하지 마십시오.

### Validation (검증)

- `csv`: 헤더가 정확하고, ID가 유일하고 안전한 상대 경로여야 하며, 필수 필드가 비어 있지 않고, 각 ID가 스킬 내부에서 확인되어야 합니다.
- 마크다운: 모든 링크가 유효하고, 엔트리가 유일하며 결정론적으로 정렬되어야 하고, 무관한 콘텐츠는 변경되지 않아야 합니다.
- 결과물에는 요청된 인덱스 모드 또는 명시적으로 요청된 조합만 포함되어야 합니다.

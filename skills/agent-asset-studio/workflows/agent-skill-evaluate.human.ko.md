# 에이전트 스킬 평가 (Evaluate Agent Skill)

## Goal (목표)

스킬을 직접 수정하지 않고, 기존 에이전트 스킬의 효과성을 평가하고 실행 가능한 조사 결과를 보고합니다.

## When to Use (사용 시점)

기존 스킬의 품질 리뷰를 수행하거나 효과성을 점검할 때 이 워크플로우를 사용합니다.

## Instructions (지침)

- 라우팅 스킬을 평가할 때만 `references/routing-skill-validation.md`를 읽으십시오.
- 사용자가 수정을 함께 요청하지 않는 한 평가 중에 파일을 직접 편집하지 마십시오.
- 사용할 수 없는 도구, 권한 또는 외부 상태가 필요한 점검 단계가 있다면 중단하고 보고하십시오.

## Workflow: 에이전트 스킬 평가 (Evaluate Agent Skill)

### Arguments from Context (컨텍스트 인자)

- 기존 스킬 경로
- 평가 범위 (더 좁은 범위가 주어지지 않은 경우 전체 품질 리뷰 수행)

### Procedure (절차)

1. `rg --files <skill-dir>`을 사용하여 대상 파일들을 나열합니다.
2. 요청된 범위를 평가하는 데 필요한 프론트매터 및 파일들만 읽습니다.
3. 트리거의 정확도, 거의 일치하지만 제외되는 항목, 출력 규약, 지침의 명확성, 컨텍스트 비용 및 리소스 경계를 점검합니다.
4. 라우팅 스킬의 경우, `references/routing-skill-validation.md`를 읽고 요청이 최소한의 정확한 워크플로우를 선택하는지 평가합니다.
5. 발견된 사항을 심각도별로 파일 위치, 증거, 최소한의 개선 조치와 함께 보고합니다. 조치할 사항이 없는 경우 이를 명시합니다.

### Validation (검증)

- 보고서는 요청된 품질 범위를 포함해야 하며, 발견된 사항과 건너뛴 점검 사항을 분리하여 표시해야 합니다.
- 각 발견 사항에는 증거, 영향도 및 최소한의 수정 사항이 포함되어야 합니다.
- 구조적 정합성 요청은 대신 `agent-skill-validate`로 라우팅되어야 합니다.

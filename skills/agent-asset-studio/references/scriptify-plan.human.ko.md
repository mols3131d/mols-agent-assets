# 에이전트 자산 스크립트화 계획 (Plan Scriptification)

## Goal

결정론적인 자연어 절차를 어떻게 스크립트로 변환할 것인지 구체적인 구현 계획(언어 선택, 저장 경로 등 포함)을 수립합니다.

## Non-Goal

- 스크립트화가 적절한지 평가하는 작업 (이는 사전에 `scriptify-evaluate.md`를 통해 수행되어야 합니다).
- 실제 스크립트를 작성하거나 수정을 실행하는 작업.

## When to Use

스크립트화가 승인된 후, 구체적으로 어떤 부분을 어떤 언어로 작성하고 어디에 저장할지 결정하기 위해 이 워크플로우를 사용합니다.

## When NOT to Use

- 해당 작업이 LLM 추론 의존성에 대해 아직 평가받지 않은 경우.

## Instructions

- 계획이 `references/zen-of-agent-assets.md` 원칙을 준수하는지 확인하십시오.
- 제안된 설계에서 추론(Reasoning)과 실행(Execution)의 명확한 분리를 유지하십시오.

## Workflow: 에이전트 자산 스크립트화 계획 (Plan Scriptification)

### Arguments from Context

- 대상 워크플로우 또는 자산 경로
- 스크립트화할 구체적인 단계나 절차
- 스크립트 작성 언어: 복수 가능, 기본값: Python
- 스크립트 저장 경로: 명시되지 않은 경우 에이전트가 문맥에 따라 판단

### Procedure

1. 스크립트화할 대상 단계를 검토합니다.
2. 문맥에 가장 적합한 스크립팅 언어를 결정합니다 (다른 언어가 명백히 더 적합하지 않은 한 기본적으로 Python 사용).
3. 스크립트의 저장 경로를 결정합니다 (예: 에이전트 스킬을 수정하는 경우 해당 스킬의 `scripts/` 디렉터리).
4. 다음 내용을 포함하는 구현 계획을 초안으로 작성합니다:
   - 제거될 정확한 자연어 지침 내용.
   - 이를 대체할 명령어 호출 구문.
   - 스크립트가 구현할 핵심 로직 요약.
5. 작성된 계획을 사용자에게 제시하고 승인을 요청합니다.

### Validation

- 계획에 스크립트 언어, 대상 경로, 영향을 받는 워크플로우 섹션이 명확히 명시되어 있습니다.
- 제안된 스크립트 로직에 LLM 추론 작업이 포함되어 있지 않습니다.

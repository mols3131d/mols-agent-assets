# 자산 마크다운 헤더 사양 (Asset Markdown Headers Specification)

에이전트 자산(Agent Asset)의 핵심 본문 구성 및 포맷 규칙입니다.

| 헤더 (Header)                   | 선택 여부 (Optional) | 설명                                                                                  |
| ------------------------------- | -------------------- | ------------------------------------------------------------------------------------- |
| **Overview**                    | ✅                   | 자산에 대한 간략한 맥락 및 배경 요약입니다.                                           |
| **Goal**                        | ❌                   | 자산의 핵심 목표 및 책임 범위입니다.                                                  |
| **Non-Goal**                    | ✅                   | 범위가 불필요하게 확장되는 것을 방지하기 위해 명시적으로 제외한 영역입니다.           |
| **When to Use**                 | ❌                   | 구체적인 시나리오, 활성화 맥락, 사용자 트리거 조건입니다.                             |
| **When NOT to Use**             | ✅                   | 제외할 시나리오 및 다른 자산으로 작업을 위임해야 하는 조건입니다.                     |
| **Instructions**                | ❌                   | 에이전트가 반드시 준수해야 하는 긍정 규칙 및 가이드라인입니다.                        |
| **Constraints**                 | ✅                   | 에이전트가 절대 위반해서는 안 되는 엄격한 부정 제약 사항입니다.                       |
| **`Workflow: <Workflow Name>`** | ✅                   | 순차적인 실행 단계 및 작업 흐름입니다. 자산이 규칙(Rules)만 포함하는 경우 생략합니다. |

## Workflow Structure (워크플로우 세부 구조)

워크플로우를 정의할 때는 `## Workflow: <Workflow Name>` 헤더를 사용하여 각 워크플로우를 구분합니다. 각 워크플로우는 다음의 하위 헤더들을 사용하여 구조화합니다.

| 헤더 (Header) | 선택 여부 (Optional) | 설명 |
| --- | --- | --- |
| **Arguments from Context** | ✅ | 컨텍스트(예: 대화 기록, 활성화된 파일, 시스템 환경 등)에서 동적으로 추출된 입력값입니다. 엄격한 스키마나 CLI 플래그 대신 시맨틱 개념이나 의도(Intents) 형태로 기술해야 하며, 이를 통해 LLM이 문맥을 유연하게 파악하고 기계적인 구조 추출에 편향되지 않도록 돕습니다. 기본값을 포함할 수 있습니다. |
| **Procedure** | ❌ | 목표 달성을 위한 순차적인 실행 단계입니다. 조건부 분기 및 종료 단계를 포함할 수 있습니다. 안전성을 위해 부정 조건 발생 시 명시적으로 조기 종료(Early termination)하는 설계를 권장합니다. 효율적인 모듈식 확장을 위해 다른 자산(규칙, 워크플로우)을 참조하거나 위임할 수 있습니다. |
| **Validation** | ✅ | 최종 결과가 올바른지 검증하기 위한 구체적인 확인 항목입니다. |
